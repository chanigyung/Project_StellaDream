Final Design Direction
----------------------

Main flow:

Player / Monster
-> WeaponInstance
-> WeaponControlBase
-> SkillContext
-> SkillExecutor
-> SkillInstance
-> SkillModule
-> SkillObjectBase / Projectile / Hitbox / Area

Core / buff extension flow:

CoreInstance / BuffManager
-> SkillModifier: numeric changes
-> SkillBehavior: behavior additions / overrides

Core concept:

SkillModifier
- Changes values through context.values.
- Examples: damage increase, cooldown reduction, projectile speed increase, hitbox size increase.

SkillBehavior
- Injects behavior into skill, weapon, projectile, or execution hooks.
- Examples: projectile bounce, area trail on movement, cast-to-charge conversion, reversed combo order.

Important design conclusion:

Skills should not contain every possible special-core behavior in advance.
Skill / Projectile / WeaponControl / SkillExecutor should expose hook points where external behaviors can intervene.
The actual special behavior should live in core behavior objects.

Responsibility example:

Projectile
- Moves.
- Detects collision.
- Gives external IProjectileBehavior a chance to handle the collision.
- If no behavior handles it, performs default destruction.

ProjectileBounceBehavior
- Checks whether bounce is possible.
- Calculates reflection angle.
- Calls Projectile.SetDirection().
- Applies speed / size changes.
- Decides whether to cancel default destruction.

Recommended Modifier / Behavior Split
-------------------------------------

SkillModifier
- Numeric change.
- Modifies context.values.
- Examples: DamageUpModifier, CooldownReduceModifier, ProjectileSpeedModifier.

SkillBehavior
- Adds or replaces behavior.
- Intervenes in hooks.
- Examples: ProjectileBounceBehavior, ReverseComboBehavior, ChargeCastBehavior.

Possible base:

public abstract class SkillModifier
{
    public virtual void OnBuildValues(SkillContext context) {}
}

Behavior marker:

public interface ISkillBehavior
{
}

Possible specialized behavior interfaces:

public interface IProjectileBehavior : ISkillBehavior
{
    bool OnProjectileCollision(Projectile projectile, Collision2D collision, SkillContext context);
}

public interface IWeaponControlBehavior : ISkillBehavior
{
    bool TrySelectSkill(...);
}

public interface ISkillExecutorBehavior : ISkillBehavior
{
    bool TryOverrideSkillFlow(SkillExecutor executor, SkillContext context);
}


Special Core Direction Examples
-------------------------------

Two one-handed weapons with the same core:
- Equipment / weapon-level core effect.
- Check in PlayerWeaponManager or core equip logic.

Effect changes against status-effected targets:
- Later, after status-effect refactor.
- Use OnHit or BeforeHit hook.

Area trail on movement skill:
- Needs IMoveSkillBehavior or movement tick/end hook.

Shorter projectile range + multi-shot:
- Use SkillRuntimeValues lifetime/speed/count or OnBeforeModuleExecute behavior.

Heal area around install object:
- Use OnObjectSpawned, check install tag, spawn heal area.

Replay skill in opposite direction after movement skill ends:
- Needs OnMoveSkillEnd or OnExpire hook.

Projectile bounce on platform + speed/size growth:
- Use IProjectileBehavior.OnProjectileCollision().
- Projectile detects collision, behavior handles reflection.

Cast skill -> charge skill:
- Needs SkillExecutor flow override. Higher difficulty.

Reverse combo order:
- Needs WeaponControlBase / ComboWeaponControl skill selection hook.

After N final combo hits, final hit auto-casts first hit:
- Needs WeaponControlBehavior + core runtime state.


Projectile Bounce Design Conclusion
-----------------------------------

Bad direction:

Projectile contains every possible bounce behavior and core only enables bounceCount.

Better direction:

Projectile detects collision and asks external behavior: "Do you want to handle this?"
Bounce angle calculation, direction change, speed/size increase, and canceling default destruction live in BounceProjectileBehavior.

Minimal Projectile API needed:

public Vector2 Direction => direction;
public int BounceCount { get; private set; }

public void SetDirection(Vector2 dir)
public void AddBounceCount()
public void MultiplySpeed(float value)

Possible collision flow:

private void OnCollisionEnter2D(Collision2D collision)
{
    if (context.behaviors.HandleProjectileCollision(this, collision, context))
        return;

    ExpireNowAndDestroy();
}
